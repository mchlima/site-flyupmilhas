<task type="auto">
  <name>Task 1: Create SectionMarketTicker component with SSR fetch and graceful fallback</name>
  <files>app/components/Section/SectionMarketTicker.vue</files>
  <action>
Create a new Nuxt 4 / Nuxt UI v4 single-file component at `app/components/Section/SectionMarketTicker.vue` that renders a slim horizontal bar displaying three quotes: Dólar (USD-BRL), Euro (EUR-BRL) and Bolsa (IBOVESPA).

Requirements:

1. **Data fetching — SSR-friendly, fresh on every open:**
   - Use two separate `useFetch` calls inside `<script setup lang="ts">` so data is available on the server render and refreshed on client navigation.
   - Currencies: `useFetch('https://economia.awesomeapi.com.br/json/last/USD-BRL,EUR-BRL', { key: 'market-fx', server: true, lazy: false })`
   - Bolsa (IBOV): `useFetch('https://brapi.dev/api/quote/%5EBVSP', { key: 'market-ibov', server: true, lazy: false })`
   - Disable Nitro caching so the user gets fresh data on each page open: pass `getCachedHeaders: () => undefined` is not needed — instead simply do NOT set `cache` and add `{ headers: { 'cache-control': 'no-store' } }` on both calls, and set `dedupe: 'defer'`. The goal is that each page load triggers a new request to the upstream API (per the constraint "atualizadas assim que o usuário abre o site").
   - Do NOT use axios. Use only Nuxt's native `useFetch` (built on ofetch / `$fetch`).

2. **Types (inline in `<script setup lang="ts">`):**
   ```ts
   interface FxEntry { bid: string; pctChange: string }
   interface AwesomeApiResponse { USDBRL: FxEntry; EURBRL: FxEntry }
   interface BrapiResult { regularMarketPrice: number; regularMarketChangePercent: number }
   interface BrapiResponse { results: BrapiResult[] }
   ```
   Cast each `useFetch` with the appropriate generic (e.g. `useFetch<AwesomeApiResponse>(...)`).

3. **Computed display values:**
   - `usd`: format `data.value?.USDBRL?.bid` as `R$ 5,12` (Brazilian format, 2 decimals, comma separator). Use `Number(bid).toLocaleString('pt-BR', { minimumFractionDigits: 2, maximumFractionDigits: 2 })` prefixed with `R$ `.
   - `eur`: same treatment for `EURBRL.bid`.
   - `ibov`: format `results[0].regularMarketPrice` as `128.456` using `toLocaleString('pt-BR', { maximumFractionDigits: 0 })` — IBOV points, no currency symbol, label it "pts".
   - For each quote, compute a `pct` number and a `pctClass`:
     - `pct >= 0` → green (`text-emerald-500`) with `▲` arrow
     - `pct < 0` → red (`text-rose-500`) with `▼` arrow
   - Format pct as `+0,24%` / `-0,34%` (Brazilian, 2 decimals, explicit sign).

4. **Loading + error handling (critical — must NOT break the page):**
   - Each `useFetch` returns `{ data, pending, error }`. Compute `showUsdEur = !pendingFx && !errorFx && data has USDBRL+EURBRL`. Same for `showIbov`.
   - If BOTH fail and both are done loading, render the whole bar with reduced markup showing "—" placeholders instead of throwing/hiding. The section root must always render so layout does not shift, unless both APIs error AND you choose to hide — but prefer placeholder dashes over hidden bar.
   - While `pending` is true, render skeleton placeholders (e.g. `<span class="inline-block w-16 h-4 bg-white/20 animate-pulse rounded" />`) in each slot to avoid CLS.
   - Wrap API calls in a try/catch via `useFetch`'s error ref — never let a rejected promise bubble.

5. **Template markup (mobile-first, matches brand):**
   ```vue
   <template>
     <section aria-label="Cotações de mercado" class="bg-[var(--color-brand-primary)] text-white border-y border-white/10">
       <div class="max-w-5xl mx-auto px-4 py-2.5 flex flex-wrap items-center justify-center gap-x-6 gap-y-2 text-xs md:text-sm font-medium">
         <!-- Dólar -->
         <div class="flex items-center gap-1.5">
           <span class="uppercase tracking-wider text-white/70">Dólar</span>
           <span v-if="pendingFx" class="inline-block w-16 h-4 bg-white/20 animate-pulse rounded" />
           <template v-else>
             <span>{{ usd }}</span>
             <span :class="usdPctClass">{{ usdArrow }} {{ usdPctLabel }}</span>
           </template>
         </div>
         <!-- Euro -->
         <div class="flex items-center gap-1.5">
           <span class="uppercase tracking-wider text-white/70">Euro</span>
           <!-- same pattern -->
         </div>
         <!-- Bolsa -->
         <div class="flex items-center gap-1.5">
           <span class="uppercase tracking-wider text-white/70">Bolsa</span>
           <!-- same pattern, label "pts" after number -->
         </div>
       </div>
     </section>
   </template>
   ```
   - Use `var(--color-brand-primary)` for background (matches AppHeader navy per project token convention — see STATE.md: "Token namespace must use --color-brand-*").
   - Text is white with 70% opacity labels. Arrows and pct colored emerald/rose.
   - Bar height compact: `py-2.5` (~40px tall). Does NOT use `py-12` or full section padding — it is a thin strip, not a full section.
   - Flex-wrap so on narrow screens items wrap to a second row rather than overflow.
   - `aria-label="Cotações de mercado"` on root `<section>` for a11y.

6. **No hardcoded hex values** — per project architecture constraint, use `var(--color-brand-primary)` for the navy background. Tailwind utility classes are fine for everything else (emerald/rose/white/opacity).

7. **Refresh strategy:** because `useFetch` runs on each SSR render AND `lazy: false` + no `cache` means the request runs on the initial page load, "os dados são atualizados assim que o usuário abre o site" is satisfied. Do NOT set up a `setInterval` — that is out of scope and would violate the spec ("assim que o usuário abre o site", not periodically).
