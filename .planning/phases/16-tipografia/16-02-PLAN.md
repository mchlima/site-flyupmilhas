<root_cause_analysis>
## Root Cause of TIPO-01 Gap

The VERIFICATION.md reported that the production CSS bundle contained `--font-family-sans:"Inter"` and that `fontRequirements.families = []` in `.nuxt/dev/index.mjs`.

**Finding after investigation:** The `fontRequirements` in `.nuxt/dev/index.mjs` belongs to `nuxt-og-image` (Satori renderer), NOT to the main @nuxt/fonts browser font module. This was a misattribution in the verification.

**Current state (fresh build):** The production CSS bundle NOW correctly shows `--font-family-sans:"Plus Jakarta Sans"` with proper fallback metrics and zero Inter references. The `_fonts/` directory contains 4 Plus Jakarta Sans woff2 files. The verification was likely run against a stale build.

**Remaining fragility:** @nuxt/fonts detects "Plus Jakarta Sans" by parsing `html { font-family: var(--font-family-sans) }` in main.css and resolving the CSS variable. This works but depends on CSS variable resolution order. If CSS import order changes or Tailwind v4 processing reorders rules, the detection could fail silently — reverting to Inter defaults.

**Fix:** Add `global: true` to force @nuxt/fonts to generate @font-face proactively (line 458-464 of @nuxt/fonts module.mjs), and add `processCSSVariables: true` to ensure CSS variable tracing works reliably.
</root_cause_analysis>

    Update the `fonts` section in nuxt.config.ts to add two critical properties:

    1. Add `global: true` to the Plus Jakarta Sans family entry. This tells @nuxt/fonts to proactively generate @font-face rules in `nuxt-fonts-global.css` without depending on CSS variable detection. From @nuxt/fonts source (module.mjs line 458): families with `global: true` get @font-face generated in a dedicated template file, bypassing the CSS-scan detection pipeline entirely.

    2. Add `processCSSVariables: true` at the top level of the `fonts` config. This enables @nuxt/fonts to follow CSS variable references like `font-family: var(--font-family-sans)` when generating fallback metrics. Without this, the font-family-injection plugin skips CSS files that only reference fonts via variables (module.mjs line 38).

    **Target state of the fonts config:**
    ```typescript

      processCSSVariables: true,
      families: [
        {
          name: 'Plus Jakarta Sans',
          provider: 'google',

          weights: [400, 500, 600, 700],
          styles: ['normal'],
        }
      ]
    },
    ```
