## Interaction Contracts

### AppHeader — Smart Sticky (D-12, D-13, D-14, D-15, D-16, D-17)

**Desktop behavior:**
- Visible at top of page; position: sticky, top: 0; z-index: 50
- On scroll DOWN past 100px: translate up (-translate-y-full) with transition `transition-transform duration-300`
- On scroll UP: translate back to 0 (translate-y-0) — "smart sticky" shows on intent
- Anchor links: `Sobre`, `Como Funciona`, `Depoimentos`, `FAQ` — call `scrollTo()` from `useScroll` composable
- CTA button: same copy as hero ("Quero minha Consultoria"), `bg-[var(--color-brand-cta)]`, scrolls to `#formulario`

**Mobile behavior (hamburger):**
- Hamburger icon: `i-heroicons-bars-3` (open state), `i-heroicons-x-mark` (close state)
- Tap opens fullscreen overlay: `fixed inset-0 bg-[var(--color-brand-primary)] z-40 flex flex-col items-center justify-center`
- Links centered vertically and horizontally in white text at 16px / weight 700 (`text-base font-bold`)
- CTA button in overlay: same orange style, full-width on mobile (`w-full max-w-xs`)
- Closes on: anchor link tap, X icon tap, Escape key
- SSR safety: hamburger open state via `ref(false)`, guarded with `import.meta.client` for any transition listener

**Scroll threshold (Claude's Discretion):** 64px — matches header height. After 64px of scroll, smart-sticky logic activates. Below 64px, header remains fully visible regardless of direction.

### Bento Grid Layout (D-04 through D-10)

**CSS Grid approach (Claude's Discretion):**
- Mobile: `grid-cols-1` — all cards stack full-width
- Tablet (md 768px): `md:grid-cols-2` — 2 equal columns, cards span 1 unless explicitly `md:col-span-2`
- Desktop (lg 1280px): `lg:grid-cols-3` — varied spans (1x1, 2x1, 1x2 pattern)
- Validate at 375px, 768px, and 1280px (per PITFALLS.md note)

**SectionMethod bento card structure:**
- Each of the 4 steps becomes a card: `rounded-xl bg-white shadow-sm p-6`
- Step 01 (Diagnóstico): standard 1x1 card
- Step 02 (Estratégia): standard 1x1 card
- Step 03 (Execução): standard 1x1 card
- Step 04 (Voo): navy accent card `bg-[var(--color-brand-primary)] text-white` for conversion emphasis
- Offer block ("O que está incluído"): spans full width below grid, keeps `rounded-xl bg-white shadow`

**SectionAbout bento card structure (Claude's Discretion):**
- Hero card (spans 2 cols on lg): company tagline + brief founding context — navy background, white text
- Value prop card 1 — Viagens executivas: `bg-white shadow-sm`, UIcon `i-heroicons-paper-airplane`
- Value prop card 2 — Economia familiar: `bg-white shadow-sm`, UIcon `i-heroicons-banknotes`
- Value prop card 3 — Renda extra com milhas: `bg-white shadow-sm`, UIcon `i-heroicons-currency-dollar` — equal billing, same card size (CONT-02)

**Hover on bento cards (Claude's Discretion):** Apply `hover:shadow-md transition-shadow duration-200` only on clickable/interactive cards. Static informational cards: no hover effect (avoids misleading interactivity cues).

### Mobile Menu Animation (Claude's Discretion)

- Overlay entry: `transition-opacity duration-200 ease-out` from `opacity-0` to `opacity-100`
- Overlay exit: reverse, `duration-150`
- No slide animation — opacity fade is SSR-safe and avoids transform reflow on mobile
- X icon replaces bars icon immediately on tap (no animation on icon swap)

## Accessibility Checklist (Phase 4 specific)

- [ ] Hamburger button has `aria-label="Abrir menu"` / `aria-label="Fechar menu"` toggling on state
